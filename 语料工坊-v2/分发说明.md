# 语料工坊分发说明

## 推荐分发方式

当前推荐使用 Windows 绿色分发包：

1. 开发者双击 `打包分发.cmd`。
2. 程序会在项目上一级目录生成 `release/语料工坊-v2-release/` 和 `release/语料工坊-v2-release.zip`。
3. 把 zip 发给使用者。
4. 使用者解压后，首次双击 `安装依赖.cmd`。
5. 依赖安装完成后，日常双击 `启动语料工坊.cmd`。

## 使用者电脑需要提前具备

- Windows 10 或 Windows 11。
- Python 3.10 或 3.11，并加入 PATH。
- Node.js LTS，并包含 npm。
- FFmpeg，并加入 PATH。

WhisperX / PyTorch 安装体积较大，首次安装耗时较长，这是正常现象。

## 分发包不包含

- `backend/.venv/`
- `backend/data/`
- `frontend/node_modules/`
- `frontend/dist/`
- WhisperX 模型缓存
- 个人音视频、数据库、备份文件、日志文件

这些内容不应该上传到 GitHub，也不应该默认发给其他用户。

## 数据保存位置

用户运行后，本地数据会保存在：

```text
backend/data/
```

其中包括：

- `backend/data/corpus.db`：数据库
- `backend/data/media/`：导入后的音视频副本
- `backend/data/work/`：临时任务目录
- `backend/data/backups/`：系统界面创建的备份

迁移数据时，优先使用系统界面的“创建备份 / 恢复备份”。

## GitHub 发布建议

如果发布到 GitHub，建议公开代码但不要上传个人数据。至少确认以下目录没有被提交：

- `backend/.venv/`
- `backend/data/`
- `frontend/node_modules/`
- `frontend/dist/`
- `release/`

如果只是给小范围用户试用，建议先使用 GitHub private repository 或直接发送分发 zip。
